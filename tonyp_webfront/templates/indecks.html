What Is This Site?

This website is not a wordpress representation of my linked in profile and resume. It does have a quick view of my previous roles and a list of skills and whatnot and if that's what you're here for keep on scrolling, you'll see it. This site IS however, much more than that. I thought it would be beneficial (and a whole lot of fun for me) to provide an actual live demonstration of some of my capabilities. The entire website and all the things directly and indirectly related is just that.
  • Front end
  • Backend
  • Security Analysis
  • Data Analytics
  • CI/CD Pipeline
  • Database
  • Provisioning Automation
  • Configuration Management Automation
  • Service Level Indicators (SLIs)/Service Level Objectives (SLOs)

This site is built in AWS. I could have done a static site generator like Pelican or Hyde, and AWS S3, but that wouldn't have been much fun. To be useful and fun for me, I chose Python and Django. Flask would have been just as fun, however, I chose Django because I wanted the "batteries included". The project is still too early on to know exactly how I want it to look. As I continue to develop, I will add more. For instance, while I was typing this up, I realized I don't have Lambda functions running and would like to include that. I added issue #21 to the Kanban board in the github I'm using to develop the project.

The entire project is being developed via Test Driven Development (TDD). I commit and run my tests locally. Once I push to github (usually to a feature branch, unless it's just html), it runs through Travis CI, where I'm testing for python 3.6 and 3.7.

There is also the security and analytics side. I have a few features you can take a look at in the project board, but, I'm dropping my ALB logs into S3 and then scraping them with my demo version of Splunk. I have built out a dashboard to track a few things:

  • Number of unique visitors
  • Their location
  • Service Level Objective
  • Not-so-normal Calls to the website
    • I use this data to block them from getting anything
As I continue to work through the project, I will probably add more features. I also created a Splunk app to that integrates with pushbullet, so I can get a daily push notifications of unique visitors to the site and immediate alerts of any significant "non-normal" activity.

You can see where I'm at with the whole project on the Kanban board. There is still a lot to be done. To be honest, I'm probably just procrastinating finishing my resume. :P
